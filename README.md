# 山のジュース屋さん — AR インタラクティブゲーム

カメラ映像（NDI 入力）をリアルタイムに解析し、子どもたちが体を動かしてジュースを作るプロジェクション AR ゲームです。

## システム構成

```
カメラ（天井・正面）
    │ NDI 映像
    ▼
[PC] main.py
    ├── ndi_receiver.py  … NDI 受信
    ├── detector.py      … 人物検出・ジャンプ計測
    ├── game_state.py    … フェーズ管理・スコア計算
    └── renderer.py      … AR 描画
    │ HDMI
    ▼
プロジェクター（床面 or スクリーン）
```

- 映像はカメラ→ NDI 送出→ PC 受信→処理→プロジェクター投影という流れです。
- プロジェクターが子どもたちの正面に向いている場合、映像は **水平反転** して出力されるため、画面内の左右が体の動きと一致します（`main.py:87` で適用）。

## ゲーム仕様

### フェーズ一覧

| # | フェーズ | 時間 | 内容 |
|---|----------|------|------|
| 1 | **IDLE** | 無制限 | 待機中。オペレーターが SPACE キーで開始 |
| 2 | **ANIMAL** | 4 秒 | 今回のリクエスト動物とその好みを表示 |
| 3 | **FRUIT_SELECT** | 15 秒 | 3 エリアに立つと対応するフルーツが降ってくる。在籍フレーム数に比例してジュース比率が決まる |
| 4 | **MIX** | 15 秒 | 全員でジャンプしてミキサーを回す。50 ジャンプで 100% 達成（早期終了あり） |
| 5 | **RESULT** | 6 秒 | 動物のリクエストとの一致度を星 1〜5 で表示 |

### エリア分割

画面を横 3 等分し、各エリアに 1 種類のフルーツを割り当てます。  
エリア境界付近（デフォルト幅の ±3%）はデッドゾーンとなり、どちらのエリアにも属しません。

### ジュース比率の計算

FRUIT_SELECT フェーズ中、各フレームで各エリアの人数をカウントし続けます。  
フェーズ終了時に累積値を合計で割ったものが最終的なジュース比率になります。

```
比率[i] = 累積人数[i] / 全エリア累積人数の合計
```

### スコア計算

動物ごとに理想の配合比率（`config.ANIMALS[*]["ideal_mix"]`）が設定されています。  
実際の比率との差をもとに一致度（0〜100%）と星評価（1〜5）を算出します。

```
差分合計 = Σ |実際の比率[i] - 理想の比率[i]|
一致度   = max(0, 1 - 差分合計 / 2)
星評価   = round(一致度 × 5)  ※最低 1 星
```

## ファイル構成

```
aeon-fruits/
├── main.py          メインループ（起動・入力・表示）
├── config.py        全パラメーター定義
├── detector.py      MOG2 背景差分 + 人物追跡 + ジャンプ検出
├── game_state.py    フェーズ遷移とスコア管理
├── renderer.py      AR 描画（パーティクル・ミキサー・星）
└── ndi_receiver.py  NDI ソース検索・接続・フレーム取得
```

## セットアップ

### 必要な環境

- Python 3.10 以上
- OpenCV (`opencv-python`)
- NumPy
- NDI を使う場合: `cyndilib`（NDI SDK が別途必要）

### インストール

```bash
pip install opencv-python numpy
# NDI 使用時
pip install cyndilib
```

### NDI SDK（NDI 使用時のみ）

NDI Tools（NewTek）の NDI SDK をインストールし、  
`cyndilib` がライブラリを見つけられる状態にしてください。

## 操作手順

### 起動

```bash
# 通常起動（NDI ソースを対話的に選択）
python main.py

# NDI ソースを番号で自動選択（例: 0 番目）
# config.py の NDI_SOURCE_INDEX = 0 を設定してから起動

# Webカメラでのデモ（NDI 不要）
python main.py --demo

# 検出デバッグ表示付きで起動
python main.py --debug
```

### NDI ソース選択（対話モード時）

起動するとネットワーク上の NDI ソース一覧が表示されます。

```
NDIソース一覧:
  1: CAMERA-01 (CAM1)
  2: SWITCHER (Program)
番号を入力してください: 1
NDI接続: CAMERA-01 (CAM1)
起動完了。SPACEキーでゲーム開始。
```

### ゲーム中のキー操作

| キー | 動作 |
|------|------|
| `SPACE` | ゲーム開始（IDLE フェーズ時のみ） |
| `d` | デバッグオーバーレイの ON/OFF 切り替え |
| `q` / `ESC` | 終了 |

### ゲームの流れ（1 ラウンド）

1. `SPACE` を押してスタート
2. **ANIMAL フェーズ**（4 秒）: 動物とリクエストを確認
3. **FRUIT_SELECT フェーズ**（15 秒）: 子どもたちが好きなフルーツのエリアに立つ
4. **MIX フェーズ**（最大 15 秒）: 全員でジャンプしてミキサーを満タンにする
5. **RESULT フェーズ**（6 秒）: 一致度と星評価を表示
6. 自動で IDLE に戻り、次のラウンドを待機

## 設定リファレンス

`config.py` で動作をカスタマイズできます。

### 映像・表示

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `WIDTH` / `HEIGHT` | 1920 / 1080 | 解像度（NDI ソースに合わせること） |
| `NDI_SOURCE_INDEX` | `None` | `None` で対話選択、整数で自動選択（0 始まり） |
| `DISPLAY_MONITOR_OFFSET_X` | `0` | プロジェクターが第 2 モニターの場合の X オフセット（例: `1920`） |

### 検出

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `DETECTION_SCALE` | `0.5` | 処理解像度の倍率（小さいほど高速・精度低） |
| `MOG2_HISTORY` | `300` | 背景モデルの学習フレーム数 |
| `MOG2_VAR_THRESHOLD` | `25` | 背景差分の閾値（大きいほど検出感度低） |
| `MIN_PERSON_BLOB_AREA` | `2000` | 人物と認識する最小ブロブ面積（検出スケールのピクセル） |
| `AREA_DEAD_ZONE` | `0.03` | エリア境界付近の不感帯（フレーム幅に対する割合） |

### ジャンプ検出

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `JUMP_Y_THRESHOLD` | `0.04` | ジャンプ判定の Y 変位（フレーム高さに対する割合） |
| `JUMP_DEBOUNCE_SEC` | `0.5` | 同一人物の連続ジャンプ計測間隔（秒） |
| `JUMPS_TO_MIX` | `50` | mix_level を 100% にするのに必要な総ジャンプ数 |

### フェーズ時間

| パラメーター | デフォルト |
|---|---|
| `PHASE_DURATIONS["ANIMAL"]` | `4` 秒 |
| `PHASE_DURATIONS["FRUIT_SELECT"]` | `15` 秒 |
| `PHASE_DURATIONS["MIX"]` | `15` 秒 |
| `PHASE_DURATIONS["RESULT"]` | `6` 秒 |

### パーティクル（フルーツ落下エフェクト）

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `FRUIT_SPAWN_RATE` | `2.0` | 満員エリアでの 1 秒あたり生成数 |
| `FRUIT_PARTICLE_SPEED` | `12` | 落下速度（ピクセル/フレーム） |
| `FRUIT_PARTICLE_RADIUS` | `28` | パーティクルの半径（ピクセル） |
| `FILL_OVERLAY_ALPHA` | `0.5` | AR オーバーレイの不透明度 |
