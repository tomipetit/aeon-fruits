# 森のジュース屋さん — AR インタラクティブゲーム

カメラ映像（NDI 入力）をリアルタイムに解析し、子どもたちが体を動かしてジュースを作るプロジェクション AR ゲームです。

## システム構成

```
カメラ（天井・正面）
    │ NDI 映像
    ▼
[Mac] main.py
    ├── ndi_receiver.py  … NDI 受信
    ├── detector.py      … YOLO 人物/顔検出・ジャンプ計測（バックグラウンドスレッド）
    ├── game_state.py    … フェーズ管理・スコア計算
    ├── ai_comment.py    … AI テイストコメント生成（Anthropic API）
    ├── renderer.py      … AR 描画（背景・スプライト・パーティクル・UI）
    └── glass.py         … コップ描画（液体色・充填アニメーション）
    │ HDMI
    ▼
プロジェクター／外部モニター
```

- 映像はカメラ → NDI 送出 → PC 受信 → 処理 → 外部モニター投影という流れです。
- プロジェクターが子どもたちの正面に向いている場合、映像は **水平反転** して出力されるため、画面内の左右が体の動きと一致します。

## ゲーム仕様

### フェーズ一覧

| # | フェーズ | 時間 | 内容 |
|---|----------|------|------|
| 1 | **IDLE** | 無制限 | 待機中。オペレーターが SPACE キーで開始。タイトル画面を表示 |
| 2 | **INTRO1** | 7 秒 | ゲーム説明 1 ページ目（初回のみ表示） |
| 3 | **INTRO2** | 10 秒 | ゲーム説明 2 ページ目（初回のみ表示） |
| 4 | **INTRO3** | 5 秒 | GAME START 画面（初回のみ表示） |
| 5 | **ANIMAL** | 4 秒 | 今回のリクエスト動物とその好みを表示 |
| 6 | **FRUIT_SELECT** | 15 秒 | 3 エリアに立つと対応するフルーツのタンクが増える。累積在籍フレーム数に比例してジュース比率が決まる |
| 7 | **MIX** | 15 秒 | 全員でジャンプしてミキサーを回す。50 ジャンプで 100% 達成（早期終了あり） |
| 8 | **POUR** | 5 秒 | 完成したジュースの色でコップが満たされるアニメーションを表示 |
| 9 | **RESULT** | 12 秒 | 動物のリクエストとの一致度を星 1〜5 で表示。AI 生成のテイストコメントを表示 |
| 10 | **ENDING** | 無制限 | 全動物終了後のエンディング画面。SPACE キーで IDLE へ戻る |

- 2 回目以降のゲームでは INTRO1〜INTRO3 はスキップして ANIMAL から始まります。
- 1 セッションにつき `ANIMALS_PER_SESSION`（デフォルト 3）体の動物を順番に処理します。

### エリア分割

画面を横 3 等分し、各エリアに 1 種類のフルーツを割り当てます。  
エリア境界付近（フレーム幅の ±3%）はデッドゾーンとなり、どちらのエリアにも属しません。

### ジュース比率の計算

FRUIT_SELECT フェーズ中、各フレームで各エリアの人数をカウントし続けます。  
フェーズ終了時に累積値を合計で割ったものが最終的なジュース比率になります。

```
比率[i] = 累積人数[i] / 全エリア累積人数の合計
```

### スコア計算

動物ごとに理想の配合比率（`config.ANIMALS[*]["ideal_mix"]`）が設定されています。  
実際の比率との差をもとに一致度（0〜100%）と星評価（0.5〜5.0）を算出します。

```
差分合計 = Σ |実際の比率[i] - 理想の比率[i]|
一致度   = max(0, 1 - 差分合計 / 2)
星評価   = round(一致度 × 10) / 2  ※最低 0.5 星
```

### 人物検出

フェーズによって使用する YOLO モデルを切り替えます。

| フェーズ | モデル | 用途 |
|---|---|---|
| FRUIT_SELECT | `yolov8n-face.pt`（顔検出） | エリア別人数カウント |
| その他 | `yolov8n.pt`（人物検出） | ジャンプ検出 |

YOLO 推論はバックグラウンドスレッドで非同期実行されるため、メインループをブロックしません。

## セットアップ

### 必要な環境

- Python 3.10 以上
- macOS（MPS 推論を使用。他 OS の場合は `detector.py` の `.to("mps")` を `.to("cpu")` に変更）

### インストール

```bash
# 仮想環境を作成して有効化
python3 -m venv venv
source venv/bin/activate

# 依存パッケージをインストール
pip install opencv-python numpy ultralytics pillow anthropic

# NDI 使用時は追加で
pip install cyndilib
```

### YOLO モデルのダウンロード

```bash
# 人物検出モデル（ultralytics が自動ダウンロード）
# 起動時に自動取得されます

# 顔検出モデル（HuggingFace から手動ダウンロード）
curl -L "https://huggingface.co/arnabdhar/YOLOv8-Face-Detection/resolve/main/model.pt" \
     -o yolov8n-face.pt
```

### API キーの設定

AI テイストコメント機能には Anthropic API キーが必要です。  
プロジェクトルートに `.env` ファイルを作成してください。

```
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
```

## HDMI 出力手順

外部モニター（プロジェクター・ディスプレイ）に全画面で出力する手順です。

### 1. モニターの配置を確認

macOS の **システム設定 → ディスプレイ → 配置** でモニターの位置を確認します。  
プライマリモニターの右にある場合、HDMI モニターの X 座標は通常 `1920`（プライマリの幅）です。

```bash
# ターミナルで解像度を確認する場合
system_profiler SPDisplaysDataType | grep Resolution
```

### 2. config.py を設定

```python
# config.py
DISPLAY_MONITOR_OFFSET_X = 1920  # HDMI モニターの X 座標
```

| 配置 | 値の例 |
|------|--------|
| HDMI モニターがプライマリの右 | `1920` |
| HDMI モニターがプライマリの左 | `-1920`（または実際の幅） |
| プライマリのみ（開発・テスト時） | `0` |

### 3. 起動

```bash
python main.py
```

`DISPLAY_MONITOR_OFFSET_X` に 0 以外の値が設定されている場合、ウィンドウが自動的に該当モニターへ移動し、**全画面表示**になります。  
`0` のままの場合はプライマリモニターにウィンドウ表示されます（開発・テスト向け）。

## 操作手順

### 起動コマンド

```bash
# 通常起動（NDI ソースを対話的に選択）
python main.py

# NDI ソースを番号で自動選択（config.py の NDI_SOURCE_INDEX に番号を設定）
python main.py

# 1 セッションで遊ぶ動物の数を指定（デフォルト: 3）
python main.py --animals 5

# Web カメラでのデモ（NDI 不要）
python main.py --demo

# カメラレスデモ（カメラ・NDI 不要）
python main.py --no-camera

# 検出デバッグ表示付きで起動
python main.py --debug
```

### NDI ソース選択（対話モード時）

起動するとネットワーク上の NDI ソース一覧が表示されます。

```
NDIソース一覧:
  1: CAMERA-01 (CAM1)
  2: SWITCHER (Program)
番号を入力してください: 1
NDI接続: CAMERA-01 (CAM1)
起動完了。SPACEキーでゲーム開始。
```

### ゲーム中のキー操作

| キー | 動作 |
|------|------|
| `SPACE` | ゲーム開始（IDLE）／IDLE へ戻る（ENDING） |
| `d` | デバッグオーバーレイの ON/OFF |
| `q` / `ESC` | 終了 |

### カメラレスデモモード（`--no-camera`）

カメラも NDI も不要で、PC 単体でゲームの全フェーズを確認できます。

- MIX フェーズはタイマーで自動進行（`DEMO_MIX_DURATION_SEC` 秒で 100% 到達）
- キーボードでジュース配合を手動調整可能

| キー | 動作 |
|------|------|
| `z` | エリア 1（左）の比率を上げる |
| `x` | エリア 2（中央）の比率を上げる |
| `c` | エリア 3（右）の比率を上げる |

## 設定リファレンス

`config.py` で動作をカスタマイズできます。

### 映像・表示

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `WIDTH` / `HEIGHT` | 1920 / 1080 | 解像度 |
| `NDI_SOURCE_INDEX` | `None` | `None` で対話選択、整数で自動選択（0 始まり） |
| `DISPLAY_MONITOR_OFFSET_X` | `0` | HDMI モニターの X オフセット。非 0 の場合そのモニターで全画面表示 |

### 検出

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `DETECTION_SCALE` | `0.5` | YOLO 処理解像度の倍率（小さいほど高速・精度低） |
| `YOLO_MODEL_PERSON` | `yolov8n.pt` | 人物検出モデル（MIX フェーズなど） |
| `YOLO_MODEL_FACE` | `yolov8n-face.pt` | 顔検出モデル（FRUIT_SELECT フェーズ） |
| `YOLO_CONF_THRESHOLD` | `0.4` | 検出信頼度の閾値 |
| `AREA_DEAD_ZONE` | `0.03` | エリア境界付近の不感帯（フレーム幅に対する割合） |

### ジャンプ検出

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `JUMP_Y_THRESHOLD` | `0.015` | ジャンプ判定の Y 変位（フレーム高さに対する割合） |
| `JUMP_DEBOUNCE_SEC` | `0.3` | 同一人物の連続ジャンプ計測間隔（秒） |
| `JUMPS_TO_MIX` | `50` | mix_level を 100% にするのに必要な総ジャンプ数 |
| `DEMO_MIX_DURATION_SEC` | `5` | `--no-camera` 時の MIX 自動進行時間（秒） |

### フェーズ時間

| パラメーター | デフォルト |
|---|---|
| `PHASE_DURATIONS["INTRO1"]` | `7` 秒 |
| `PHASE_DURATIONS["INTRO2"]` | `10` 秒 |
| `PHASE_DURATIONS["INTRO3"]` | `5` 秒 |
| `PHASE_DURATIONS["ANIMAL"]` | `4` 秒 |
| `PHASE_DURATIONS["FRUIT_SELECT"]` | `15` 秒 |
| `PHASE_DURATIONS["MIX"]` | `15` 秒 |
| `PHASE_DURATIONS["POUR"]` | `5` 秒 |
| `PHASE_DURATIONS["RESULT"]` | `12` 秒 |

### UI・演出

| パラメーター | デフォルト | 説明 |
|---|---|---|
| `TYPEWRITER_CHARS_PER_SEC` | `10` | タイプライター演出の速度（文字/秒） |
| `ANIMALS_PER_SESSION` | `3` | 1 セッションで登場する動物の数 |
| `FRUIT_SPAWN_RATE` | `2.0` | 満員エリアでの 1 秒あたりパーティクル生成数 |
| `FRUIT_PARTICLE_SPEED` | `12` | フルーツ落下速度（ピクセル/フレーム） |
| `FRUIT_PARTICLE_RADIUS` | `28` | フルーツパーティクルの半径（ピクセル） |
